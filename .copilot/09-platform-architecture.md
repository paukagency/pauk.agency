# 09 — Platform Architecture

## Роли пользователей

| Роль | Описание | Дашборд |
|------|----------|---------|
| **Инфлюенсер** | Продвигает приложения через свой промокод | `dashboard-influencer.html` |
| **Администратор (Pauk.Agency)** | Полный доступ к управлению платформой | `dashboard-admin.html` |
| **Разработчик приложения** | Сторонний девелопер, статистика своих приложений | `developers.pauk.agency` (отдельный субдомен, позже) |

## Что видит каждая роль

### Инфлюенсер
- Свои промокоды по каждому приложению
- Статистика: установки, доход, клики
- Выплаты и история транзакций
- Настройки профиля

### Разработчик приложения (сторонний)
- Статистика по своим приложениям: установки через инфлюенсеров, расходы на выплаты
- Список активных инфлюенсеров, продвигающих их приложение
- Настройки промо-условий (комиссия, скидка, срок)
- **НЕТ** доступа к другим приложениям и управлению платформой

### Администратор (Pauk.Agency)
- Всё вышеперечисленное
- Управление всеми пользователями (инфлюенсеры, разработчики)
- Управление всеми приложениями на платформе
- Финансовые отчёты и настройки платформы
- Модерация: одобрение/отклонение заявок

## Приложения на платформе

### Типы приложений
1. **Собственные** (Pauk.Agency) — 2–3 штуки для старта
2. **Партнёрские** (сторонние разработчики) — добавляются по мере роста

### app-detail страница
- Один HTML-шаблон `pages/app-detail.html`
- Данные подставляются через JS при переходе (URL-параметр `?app=slug` или отдельные страницы-копии на старте)
- **Цветовая тема специфична для каждого приложения**: акцентный цвет иконки, плейсхолдера скриншота и некоторых блоков берётся из palette приложения
- Нейтральные элементы (promo-block, stats, features) остаются в стандартных переменных

### Цветовая тема приложения
Каждое приложение имеет свою пару цветов:
```js
// Пример в данных приложения
{
  slug: "appname-one",
  colorA: "#d4f542",   // светлый (начало градиента иконки, плейсхолдер)
  colorB: "#a8e000",   // тёмный (конец градиента)
}
```
Применяется через CSS custom properties на элементе-обёртке или инлайново (`style="background: linear-gradient(135deg, colorA, colorB)"`).

## Страницы сайта (основной домен)

```
/index.html                        # лендинг
/pages/apps.html                   # каталог приложений
/pages/app-detail.html             # шаблон страницы приложения
/pages/login.html                  # вход
/pages/dashboard-influencer.html   # ЛК инфлюенсера
/pages/dashboard-admin.html        # ЛК администратора
```

> ЛК разработчиков приложений — отдельный субдомен `developers.pauk.agency`, вне этого репозитория.

## Приоритет разработки
1. Публичные страницы (index, apps, app-detail) — сначала
2. Авторизация (login) — после контента
3. ЛК инфлюенсера — MVP
4. ЛК администратора — после запуска
5. ЛК разработчиков — отдельный субдомен, отдельный этап

## Важно при разработке дашбордов
- Разные роли = разные сайдбары (разные nav-items)
- Общие компоненты (stat-card, таблицы) — переиспользуемые CSS-классы
- Mock через `localStorage` → потом Supabase (см. `10-data-backend.md`)
- Безопасность на фронте — только UI-разделение; реальная авторизация через Supabase Auth
